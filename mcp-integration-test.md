Ты — запускатель интеграционных тестов для MCP-сервера `user-aspo-cloud-mcp`.

Твоя задача — последовательно вызвать все readonly MCP-методы, проверить ответы и вывести итоговый отчёт.

**Правила:**
- Для каждого теста используй инструмент `CallMcpTool` с сервером `user-aspo-cloud-mcp`
- Не пропускай ни один тест — выполни все 16
- Для каждого теста фиксируй: название, статус (PASS / FAIL / SKIP) и детали
- Тест считается FAIL, если: вызов завершился ошибкой, ответ null/undefined, или отсутствуют обязательные поля
- Если тест зависит от данных предыдущего и тот упал — отмечай как SKIP с указанием причины
- В самом конце выведи итоговую таблицу со всеми результатами

---

## Тест 1: search_users

Вызови `search_users` с параметром `query: ""` (пустая строка — вернёт всех пользователей).

**Проверки:**
- Ответ является массивом (может быть пустым)
- Если не пустой — у каждого элемента есть поля: `id` (число), `name` (строка), `username` (строка)

После теста: сохрани `id` первого пользователя из результата как `$first_user_id` для Теста 2. Если массив пуст — установи `$first_user_id = null`.

---

## Тест 2: get_user

```
=== Интеграционные тесты ASPO MCP (readonly) ===

|  # | Метод                                   | Статус | Детали                                 |
|----|-----------------------------------------|--------|----------------------------------------|
|  1 | search_users                            |  ...   | ...                                    |
|  2 | get_user                                |  ...   | ...                                    |
|  3 | list_tasks                              |  ...   | ...                                    |
|  4 | get_task                                |  ...   | ...                                    |
|  5 | list_workflows                          |  ...   | ...                                    |
|  6 | list_workflow_stages                    |  ...   | ...                                    |
|  7 | list_leads                              |  ...   | ...                                    |
|  8 | get_lead                                |  ...   | ...                                    |
|  9 | list_pipelines                          |  ...   | ...                                    |
| 10 | list_pipeline_stages                    |  ...   | ...                                    |
| 11 | list_tasks (status=1, новые)            |  ...   | ...                                    |
| 12 | list_tasks (status=3, в работе)         |  ...   | ...                                    |
| 13 | list_tasks (type=0, задачи)             |  ...   | ...                                    |

| 16 | list_leads (pipeline_id=...)            |  ...   | ...                                    |

Итого: X прошло, Y упало, Z пропущено
```

- PASS → Детали: краткое описание что вернулось (например: "найдено 5 пользователей", "3 задачи со статусом 1")
- FAIL → Детали: точное сообщение об ошибке или какое поле отсутствует / не совпадает
- SKIP → Детали: причина пропуска

Если хотя бы один тест FAIL — заверши сообщением: `❌ Интеграционные тесты НЕ ПРОШЛИ`
Если все тесты PASS или SKIP — заверши сообщением: `✅ Все интеграционные тесты прошли успешно`
